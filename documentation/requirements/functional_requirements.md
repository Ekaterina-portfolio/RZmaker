# **Описание функциональных требований: Для Фазы 1 (MVP)**

### **FR-01:**

**Только гостевой режим:**

- Система идентифицирует пользователей по уникальному идентификатору сессии
- Временный идентификатор сессии, создается при первом входе пользователя в систему и хранится в браузере
- Генерируется уникальный ID сессии на стороне сервера и привязывается к действиям пользователя в текущей сессии

### **FR-02:**

Заполнение профиля пользователя включает в себя требования: FR-02_01, FR-02_02, FR-02_03, FR-02_04

**FR-02_01:**

Система должна предоставлять веб-форму для ввода данных

**FR-02_02:**

**Форма должна содержать поля:**

- Имя (обязательное)
- Фамилия(опционально)
- Фото (опционально)
- Пол (обязательно)
- Город поиска работы(обязательно)
- Дата рождения (опционально)
- Профессия (обязательное, выбор из списка - привязан к 3D-моделям)
- Образование и курсы (обязательно)
- Навыки(Обязательно,выпадающий список, реализовано в виде тегов. Поддерживает добавление пользовательских тегов)
- Опыт работы (опционально)
- Текст “О себе" (опционально)
- Знание языков (опционально)
- Пожелания по графику и формату работы (опционально)
- Готовность к командировкам и переезду (опционально)
- Контактные данные: телефон (опционально), почта (обязательно), соцсети (опционально)
- Предпочитаемый канал связи (опционально)

**FR-02_03:**

**Система должна выполнять валидацию форматов:**

- Email: проверка формата email@domain.com
- Телефон: проверка формата +7 123 456 78 90 или +375 12 345 67 89
- Дата рождения: корректность даты (от 16 до 100)
- Проверка дублирования данных

**Визуальный фидбек:**

- Подсветка невалидных полей красным
- Сообщения об ошибках под полями
- Форма блокируется до исправления ошибок

**FR-02_04:** 

Рядом с полем выбора профессии система должна в реальном времени отображать предварительный просмотр 3D-персонажа, соответствующего выбранной профессии**

### **FR-03:**

Управление данными включает в себя требования: FR-03_01, FR-03_02, FR-03_03, FR-03_04

**FR-03_01:**

После успешной валидации система должна сохранять данные профиля во временное хранилище (во временной таблице в базе данных)

**FR-03_02:**

Срок временного хранения 24 часа

**FR-03_03:**

После сохранения во временное хранилище данных система должна автоматически запускать процесс генерации резюме

**FR-03_04:**

После подтверждения генерации данные из временной таблицы переносятся в базу данных и через 3 месяца удаляются автоматически

### **FR-04:**

Генерация резюме включает в себя требования: FR-04_01, FR-04_02, FR-04_03, FR-04_04, FR-04_05, FR-04_06, FR-04_07

**FR-04_01:**

Система должна генерировать резюме на основе введенной информации

**FR-04_02:**

Сгенерированное резюме должно иметь уникальный, профессиональный дизайн

**FR-04_03:**

После генерации пользователю показан предпросмотр для проверки корректности

**FR-04_04:**

Пользователь после генерации может вернуться на шаг назад чтобы откорректировать данные и снова запустить генерацию

**FR-04_05:**

Пользователь может отменить генерацию

**FR-04_06:**

Пользователь может подтвердить генерацию резюме и перейти к оплате(обработка оплаты после добавления монетизации)

**FR-04_07:**

В базе данных есть таблица для хранения шаблонов (обработка после добавления монетизации)

### **FR-05:**

Оформление сайта включает в себя требования: FR-05_01, FR-05_02, FR-05_03, FR-05_04, FR-05_05

**FR-05_01:**

**Главная страница**

- Должна быть картинка-анимация с героем за компьютером, выполняющим разлиные действия(машет, печатает, двигает мышкой)
- Должна быть адаптирована под мобильную версию
- Должна содержать кнопку "Создать резюме"
- Должно быть приветствие пользователя

**FR-05_02:**

Блок "О нас" должен содержать:

- Описание компании
- Преимущества сервиса
- Ключевые особенности

**FR-05_03:**

Блок "Образец резюме" (Две страницы) должен содержать:

- Страница 1: Фото, имя, фамилия, контакты, должность, навыки и опыт работы - слева, а персонаж и указатель прокрутки вниз - справа
- Страница 2: Опыт работы, О себе - слева, а персонаж - справа


**FR-05_04:**

**Блок “Отзывы” соответствует требованиям:**

1. Имеет форму для отзыва с полями:
- Email
- Оценки (Выбор от 1 до 5)
- Текст отзыва

2. После выбора оценки и\или написания текста отзыва пользователь сохраняет свой отзыв

3. Отзыв хранится в базе данных

**FR-05_05:** 

**Блок "Контакты" содержит:**

- Номер телефона
- Email
- Telegram

### **FR-06:**

Анимации включают в себя требования: FR-06_01, FR-06_02

**FR-06_01:**
Система должна содержать минимум одну 3D-модель с анимацией (Например: программист сидит за компьютером и печатает, двигает мышкой)

**FR-06_02:**

**Отображение в резюме:**

- Сгенерированное резюме должно включать 3D-модель
- Персонаж должен быть статичным для выгрузки в PDF/Word или иметь анимацию для представления в виде лендинга
- Пользователь может выбрать выгрузку резюме  в PDF/Word  без анимаций и 3D-моделей

### **FR-07:**
Форматы вывода резюме включают в себя требования: FR-07_01, FR-07_02, FR-07_03, FR-07_04

**FR-07_01:** 

**Поддержка форматов:**

- Система должна генерировать ссылку на лендинг, который позволяет выгрузить резюме в формате Word/PDF
- Лендинг должен быть оптимизирован для печати

**FR-07_02:**

**Выбор формата:**

- После генерации резюме и перехода по ссылке пользователь должен иметь возможность:
- Скачать резюме в PDF
- Скачать резюме в DOCX
- Открыть версию для печати в браузере

**FR-07_03:**

**Требования к PDF:**

- PDF-файл должен сохранять все элементы дизайна (шрифты, цвета, 3D-персонаж в статичном виде)
- Пользователь может выбрать выгрузку без 3D-персонажа и анимаций
- Размер страницы - стандартный А4

**FR-07_04:**

**Требования к печати:**

- Автоматическая разметка под печать (поля, ориентация)
- При печати из браузера резюме должно корректно отображаться на бумаге
- Отсутствие обрезанных элементов на границах страницы
- Черно-белая печать должна сохранять читаемость
